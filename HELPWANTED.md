# Help Wanted

We welcome contributions from the community! Here are some areas where you can get involved with HAPI FHIR.

## Getting Started

If you're new to the project, start by reading the [Contributors Guide](CONTRIBUTORS.md) and the [Hacking Guide](hapi-fhir-docs/src/main/resources/ca/uhn/hapi/fhir/docs/contributing/hacking_guide.md).

## Contribution Areas

### 📘 Documentation
- Improve or update existing documentation in `hapi-fhir-docs/`
- Add code examples for common use cases (Patient CRUD, search queries, subscriptions)
- Translate or proofread documentation for clarity

### 🧪 Testing
- Increase test coverage for modules with low coverage
- Add integration tests for FHIR R4/R5 operations
- Create test cases for edge cases in search parameter handling

### 🐛 Bug Fixes
- Check the [open issues](https://github.com/hapifhir/hapi-fhir/issues) for bugs marked `bug`
- Issues with fewer comments are often simpler to tackle

### ✨ Feature Ideas
- Investigate adding support for FHIR's RDF (Turtle) encoding to HAPI
- Improve CDS Hooks integration and expand supported hook types
- Enhance Bulk Data Export for large-scale data operations

### 🔧 Tooling & CI
- Improve build speed and CI pipeline performance
- Update deprecated dependencies and resolve compiler warnings
- Add static analysis or linting tools for code quality

## How to Claim an Issue

1. Comment on the issue to let maintainers know you'd like to work on it
2. Fork the repository and create a feature branch
3. Submit a pull request referencing the issue number

## Questions?

See the [Getting Help](https://github.com/hapifhir/hapi-fhir/wiki/Getting-Help) wiki page for support channels.
